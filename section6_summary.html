📋 מה זה Section 6?

סעיף 6 דורש שימוש בפקודת aggregate() עם כל הארגומנטים הבאים: $match, $project, $unwind, $group, $lookup, $sort, $out, ועוד.

הפרויקט הזה מיישם את כולם בתוך מערכת אמיתית — בונה PC שבוחר 8 רכיבים תואמים בתקציב נתון.

⚙️ Aggregate Stages — איפה כל אחד?

טבלה מלאה של כל הארגומנטים של aggregate() ואיפה הם מופיעים בקוד:

Stage מיקום מה עושה
$match stepCPU, stepMotherboard, getTierBoundaries, finalize, 3 pipelines סינון ראשוני — type, price, socket
$project stepCPU, stepMotherboard, stepCase, getTierBoundaries, finalize, 2 pipes בחירת שדות + חישובים (computed fields)
$lookup stepMotherboard — Self-Join CPU → Motherboard לפי socket (מאותה קולקציה!)
$unwind stepMotherboard שטיחת מערך התוצאות של $lookup
$group getTierBoundaries, getMinCosts, marketAnalysis, manufacturerBreakdown צבירה — $sum, $avg, $min, $max, $push, $first
$sort כל שלב מיון לפי מחיר / score
$limit כל שלב הגבלה ל-15/20/30 תוצאות
$out finalizeBuild שמירת המחשב לקולקציית recommended_combos

אופרטורים מתמטיים (בתוך $project)

אופרטור מיקום מה עושה
$add finalizeBuild סכום כל מחירי הרכיבים
$subtract stepCase חישוב GPU Clearance (פנוי במארז)
$multiply getTierBoundaries, marketAnalysis חישוב אחוזונים ושווי שוק
$expr stepMotherboard, stepCase השוואה בין שדות מחושבים

🔗 Self-Join — הלב של Section 6

הפיצ'ר המרכזי: $lookup על אותה קולקציה (components → components).

stepMotherboard — Self-Join Pipeline
// Stage 1: מצא את ה-CPU שנבחר
{ $match: { _id: cpu._id } }

// Stage 2: Self-Join — חפש רכיבים עם socket תואם
{ $lookup: {
    from: "components",        // אותה קולקציה!
    localField: "requirements.socket_match",
    foreignField: "specs.socket",
    as: "socket_matches"
}}

// Stage 3: שטח מערך לשורות
{ $unwind: "$socket_matches" }

// Stage 4: סנן — רק לוחות-אם בטווח מחיר
{ $match: { "socket_matches.type": "Motherboard", ... } }

// Stage 5: עצב מחדש
{ $project: { name: "$socket_matches.name", ... } }

// Stage 6-7: מיון + הגבלה
{ $sort: { price: 1 } }, { $limit: 30 }

🏷️ מערכת Tier — איזון רכיבים חכם

מונעת שילובים אבסורדיים (i9 על לוח H610, מארז ב-$55 על build ב-$10,000).

Entry (0–25%)
Mid (25–60%)
High (60–85%)
Enthusiast (85%+)
getTierBoundaries() — Aggregate Pipeline
// חישוב גבולות Tier לפי אחוזוני מחיר בDB
{ $match: { type: "CPU", price: { $type: "number" } } }
{ $sort: { price: 1 } }
{ $group: {
    _id: null,
    prices: { $push: "$price" },  // כל המחירים למערך
    total: { $sum: 1 }             // כמה רכיבים
}}
{ $project: {
    entry_max: { $arrayElemAt: ["$prices",
        { $floor: { $multiply: ["$total", 0.25] } }] },
    mid_max:   { $arrayElemAt: ["$prices",
        { $floor: { $multiply: ["$total", 0.60] } }] },
    high_max:  { $arrayElemAt: ["$prices",
        { $floor: { $multiply: ["$total", 0.85] } }] }
}}

חוקי Tier

🔹 Mobo חייב להיות ±1 Tier מה-CPU

🔹 PSU חייב להיות ±1 Tier מה-GPU

🔹 RAM, Storage, Cooler, Case — min tier = buildTier - 1

🧩 Section 3 — פונקציות JavaScript

הסעיף דורש פונקציות JS מוגדרות-משתמש. הנה כולן:

פונקציה תפקיד
getProfileParams(usage) מחזיר פרמטרים לפי פרופיל: gaming / workstation / budget / enthusiast
calculateWeightedScore() ציון ביצועים משוקלל — GPU*0.6 + CPU*0.4 (gaming)
calculateRequiredWatts() חישוב ספק כוח נדרש (TDP + VRAM × 20 + 25%)
getCoolerReserve() שריון מינימלי למקרר לפי TDP אפקטיבי
caseSupportsMotherboard() Full Tower > ATX > MicroATX > Mini ITX
getTierBoundaries(type) גבולות Tier מ-aggregate על מחירים (Section 6)
priceTier(price, bounds) ממפה מחיר → Tier (1-4)
tierToFloor(minTier, bounds) ממפה Tier מינימלי → מחיר רצפה
getMinCosts() aggregate + $min — מחיר מינימלי לכל סוג רכיב
futureReserve(types) שריון תקציב לרכיבים עתידיים (Quality-Aware)
validateAndSave() בדיקת בחירה + הגנת תקציב (max 110%)
printComponentList() הדפסה מעוצבת עם עמודות דינמיות

🧠 פיצ'רים חכמים

✅ מומש

Relevance Filter

GPU עם score < 1000 נחסמים — מונע בחירת כרטיסים עתיקים (GT 730, FirePro 2014).

✅ מומש

Surplus-Aware GPU

אם CPU+Mobo+RAM יצאו זול מהצפוי, 70% מהחיסכון עובר ל-GPU. 30% נשמר כבאפר.

✅ מומש

RAM Hoarder Fix

maxRamGb בפרופיל — gaming מוגבל ל-32GB. מיון price:1 במקום capacity:-1.

✅ מומש

Build-Tier Quality

CPU Tier 4 (Enthusiast) → כל רכיב חייב Tier 3+. מונע RAM ב-$43 על build ב-$10k.

⚠ ידוע

Cooler/Case Mismatch

360mm cooler נבחר לפני Case (שלב 6 vs 8). ATX mobo לא מפעיל פילטר.

⚠ ידוע

Glass Cannon

אלגוריתם חמדן ממקסם CPU+GPU score, לא "חוויה" מאוזנת.
